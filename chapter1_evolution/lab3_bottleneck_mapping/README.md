# Lab 3: Bottleneck Mapping

## Objective
Identify bottlenecks in a software delivery pipeline.

## Activity
- Sketch a simple delivery pipeline: Code → Build → Test → Deploy → Monitor.
- Mark potential bottlenecks (e.g., slow testing, manual approvals).
- Brainstorm automation or collaboration improvements.

## Reflection Questions
- Which stage slows down delivery the most?
- How can automation or better collaboration reduce bottlenecks?
- How do bottlenecks affect deployment frequency?

## Case Study Reference
- LinkedIn mapped bottlenecks in their release process and reduced lead time by automating testing.

## Notes
Conceptual mapping exercise — use diagrams or whiteboard.
